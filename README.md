##  Технічна складова та архітектура
При розробці проєкту було реалізовано ключові вимоги до сучасної фронтенд-розробки та семантики:

*   **HTML5 (Семантична структура):** Чиста розмітка сторінок із використанням правильних тегів семантики (`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`).
*   **CSS3 (Адаптивність, Змінні та Анімація):**
    *   **Flexbox & CSS Grid:** Навігаційне меню та хедер побудовані на Flexbox. Сітка карток головних персонажів реалізована за допомогою CSS Grid, що робить інтерфейс повністю адаптивним під будь-які екрани.
    *   **CSS-змінні (:root):** Налаштовано змінні кольорів, що дозволяє легко перемикати дизайн сайту між світлою (`.light-theme`) та темною темами.
    *   **Анімації (@keyframes):** Реалізовано плавну анімацію появи контенту при завантаженні сторінок (`smoothAppearance`).
*   **JavaScript & jQuery:** Логіка інтерфейсу поєднує використання чистих методів JS для роботи з DOM та можливостей бібліотеки jQuery для створення динамічних елементів без перезавантаження сторінок.

##  Реалізований інтерактивний функціонал

1.  **SPA-навігація (Система Табів на jQuery):** Перемикання між основними інформаційними вкладками сайту відбувається миттєво всередині однієї сторінки за допомогою класів `.tab-link` та `.tab-item`. Активна вкладка підсвічується фірмовим блакитним кольором CyberLife.
2.  **Динамічний конструктор елементів (Робота з DOM):** Інтегровано інтерактивну секцію керування модулями програмного забезпечення андроїдів. Користувач може в реальному часі вводити назву модуля, додавати його до списку та видаляти елементи за допомогою кнопки `.delete-btn` на чистому JavaScript.
3.  **Імітація AJAX-запиту (Форма та Валідація):**
    *   Форма зворотного зв'язку містить клієнтську валідацію полів на наявність помилок (виводиться текст `.error-text`).
    *   При успішному заповненні імітується асинхронна затримка відповіді сервера (`setTimeout`), після чого відкривається модальне вікно (`.modal-overlay`) з анімованим індикатором статусу девіантconfiguration.
4.  **Кастомізація аватарок персонажів:** Для карток Коннора, Маркуса та Кери налаштовано відображення реальних зображень через CSS-властивість `background-size: cover` з ефектом індивідуального неонового світіння їхніх діодів.
